# EBLAN Browser — полный чек-лист функций + бэкенд + сравнение

> Документ собран по факту из кода (`EBLAN_DEBUG.py`, `backend/`), а не по обещаниям.
> Легенда: ✅ есть · 🟡 частично/упрощённо · 🃏 шуточная/тролль-фича · ❌ нет · ⚠️ проблема/риск

Движок: **PyQt6 QtWebEngine = Chromium (Blink + V8)**. То есть по рендерингу EBLAN —
*Chromium-подобный* браузер (как Chrome/Brave/Edge), а **не** Gecko (Firefox).
Архитектурно — однофайловое Python-приложение поверх Qt, не форк Chromium.

### Где EBLAN выигрывает
- Встроенный VLESS/Xray-клиент «из коробки» (у мейнстрима, кроме Brave/Opera, нет).
- Жёсткий контроль доступа: вайтлист доменов + бан по железу с нечётким совпадением — нишево, но мощно для «закрытого» сценария.
- Лёгкая кастомизация UI через Ecss и свой формат расширений.

### Где EBLAN проигрывает (фундаментально)
- Нет закладок, поиска по странице, зума, менеджера паролей, автозаполнения — базовый UX.
- Нет экосистемы WebExtensions, полноценных DevTools, мобильной версии.

- Не самостоятельный браузер, а обёртка над QtWebEngine: обновления безопасности
  Chromium зависят от версии Qt, нет своего sandbox-менеджмента.
- ⚠️ Захардкоженные секреты (Mistral key, креды бэкенда).

### Резюме
EBLAN — **нишевый Chromium-обёрточный браузер** с упором на *контроль* (встроенный
VPN, вайтлист, бан по железу, серверная админка) и набором шуточных фич
(FPS «1488», счётчик проёба). По базовому браузерному UX и приватности он
заметно позади Firefox/Chrome/Brave, но в своей нише (управляемый «закрытый»
браузер с VPN и анти-обходом банов) делает то, чего у них нет.
